# Changelog

## [0.1.15] - 2026-06-13

- 서버 자원 고갈 완화를 위해 `Summary:EnableGeneration` 설정을 추가하고 기본값을 비활성화
- RSS 수집 파이프라인은 유지하되 자동 전날/주간 요약 생성은 설정이 켜진 경우에만 실행하도록 변경
- 관리자 요약 재생성, 미리보기, force 요약 API가 요약 생성 비활성화 상태에서는 409 응답을 반환하도록 보호

## [0.1.14] - 2026-06-10

- 주간 요약 통계 카드의 기준 기간이 명확히 보이도록 완료 주간 날짜 범위를 표시
- 동일 app.js 캐시 URL 재사용을 피하기 위해 프론트엔드 캐시 버스터를 0.1.14로 갱신


## [0.1.13] - 2026-06-10

- 이슈 요약 화면의 주간 통계가 최근 7일 뉴스 검색 데이터와 저장 주간 요약 데이터를 섞어 표시하던 문제 수정
- 전날 요약 관련 기사 매칭을 전날 날짜 범위로 제한하고 주간 요약은 저장된 완료 주간 범위를 기준으로 관련 기사와 통계를 계산하도록 정리
- 저장 주간 요약의 카테고리별 이슈 수와 기사 수가 있으면 해당 값을 우선 표시해 요약 문구와 통계 카드 기준을 맞춤


## [0.1.12] - 2026-06-10

- 전날/주간 요약 관련 기사 조회 대상을 선택 카테고리 내부로 제한하지 않고 전체 기사 그룹에서 articleIds 기준으로 매칭하도록 수정
- 카테고리를 넘나드는 통합 이슈에서도 표시 기사 수와 실제 관련 링크 목록이 일치하도록 개선


## [0.1.11] - 2026-06-10

- app.js 캐시 버스터를 배포 버전 기준으로 갱신해 브라우저와 Cloudflare가 이전 프론트엔드 코드를 재사용하지 않도록 수정
- 릴리스 스크립트가 index.html의 app.js 버전 쿼리도 자동으로 갱신하도록 개선


## [0.1.10] - 2026-06-10

- 전날/주간 요약의 통합 이슈 관련 기사 링크를 첫 번째 매칭 그룹에서 멈추지 않고 모든 articleIds 매칭 그룹에서 병합하도록 수정
- 통합 이슈의 표시 기사 수와 실제 관련 링크 목록이 어긋나지 않도록 중복 URL 제거 후 링크 목록 구성


## [0.1.9] - 2026-06-10

- 전날 요약 대표 이슈 선별 시 반복 키워드로 쪼개진 관련 이슈를 통합 후보로 재집계
- 통합 이슈의 기사 수·출처 수·반복 키워드를 반영해 투표용지 부족 사태처럼 여러 후속 기사로 확산되는 이슈가 주요 요약에 올라오도록 개선


## [0.1.8] - 2026-06-10

- 기존 저장 요약을 덮어쓰지 않고 새 전날 요약을 생성하는 관리자 preview API 추가
- 비교용 preview 응답은 `saved=false`로 반환하고 MongoDB `summaries` 컬렉션에는 저장하지 않도록 처리

## [0.1.7] - 2026-06-10

- 전날 요약에서 카테고리별 키워드 분포, 출처 수, 기사 수, 중요도 점수를 함께 반영해 OpenAI 입력 후보를 선별하도록 변경
- OpenAI 전날 요약 입력에 대표 기사 제목, RSS 요약, 수집 본문 일부, 주요 키워드를 포함하도록 개선
- 주간 요약은 OpenAI를 호출하지 않고 완료 주간에 해당하는 일간 요약들을 기반으로 로컬 집계 생성하도록 변경
- OpenAI 입력에 사용하는 대표 기사 근거는 저장소와 공개 API 응답에 포함하지 않도록 처리

## [0.1.6] - 2026-06-10

- 공개 요약 API가 사이트 접속 시 OpenAI 요약을 생성하지 않고 저장된 요약만 반환하도록 변경
- Collector가 전날 요약과 최근 완료 주간 요약을 저장값이 없을 때만 생성하도록 수정
- 주간 요약 기준을 한국 시간 월요일 00시 이후 직전 월요일부터 일요일까지의 완료 주간으로 정리

## [0.1.5] - 2026-06-10

- 과거 RSS 기사도 현재 활성 언론사 기준으로 뉴스 검색에 노출되도록 복원
- RSS URL 정확히 일치 기준에 언론사 매핑 기준을 추가해 과거 기사 표시 범위 확대

## [0.1.4] - 2026-06-10

- 푸터에서 불필요한 로컬 테스트 문구 제거

## [0.1.3] - 2026-06-10

- 서비스 고지에서 개인용 모니터링 도구 표현을 공개 뉴스 브리핑 서비스 기준으로 수정
- 푸터 서비스 설명 문구를 RSS 기반 뉴스 브리핑 서비스로 정리

## [0.1.2] - 2026-06-10

- 웹 서버와 독립 실행되는 Collector 작업 스케줄러 배포 추가
- Collector 운영 배포 스크립트와 자동 재시작 실행 스크립트 추가

## [0.1.1] - 2026-06-10

- 사이트 푸터에 현재 배포 버전 표시

## [0.1.0] - 2026-06-10

- Pulse Brief의 첫 공식 버전 기준을 추가했습니다.
- RSS 수집, MongoDB 저장, 관리자 콘솔, OpenAI 요약, Cloudflare Tunnel 배포 구성을 포함합니다.
- 웹 서버와 수집기 분리 구조를 포함합니다.
